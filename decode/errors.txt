Для файлов .tonecomp

Метки реф. разметки:
e - тон есть в реф.
o - тона нет в реф.

Метки тестовой разметки:
e - тон есть в тест. и не был отбракован
o - тона нет в тест.
b - тон был отбракован.

Причины браковки (тестовая разметка):

Применить к числовому коду ошибки операцию битового И (можно использовать my_bitand из helpfunc.jl)
со значениями 2^n, где n - значение из словаря кодировок (см. ниже) для файлов tone или pres.
При ненулевом результате применения битового И, причина браковки соответвует расшифровке n для
соответсвующего файла (возможно несколько ненулевых результатов - несколько причин).

(Удобнее воспользоваться функцией errorcode из helpfunc.jl, подав на вход код ошибки (bad из разметки)
и максимальное значение n из таблицы кодировок для соответсвующего файла (см. ниже)).

Словарь кодировок причин браковки.
2^n, где n:

(для tone)
0 - давление в точке меньше 30 мм
1 - сигнал/шум < 3
2 - амплитуда тона < 0.3

(для pres)
0 - ширина ДО (~RR) ВНЕ диапазона 250 - 1500 мс
1 - время вронта меньше 80 мс
2 - абсолютная амплитуда (размах) ВНЕ диапазона 1 - 50 мм
3 - коэффициент скважности ВНЕ диапазона 0.1-0.8
4 - преждевременная пульсация (отношение интервалов До/После меньше 0.5, интервал До меньше 500 мс)
5 - широкая пульсация (отношение интервалов До/После больше 2, интервал После больше 1000 мс)
6 - скорость накачки в пульсации ВНЕ диапазона -100 - 100 
7 - отношение скоростей накачки Текущий/Предыдущий больше 20 (широкая?)
